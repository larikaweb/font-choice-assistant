# Схема RAG-chunk для базы подбора шрифтов

## Назначение

Этот файл описывает структуру RAG-базы для AI-ассистента подбора шрифтовых пар.

Полный skill не публикуется. В RAG-базу попадает только безопасный слой:
- табличные знания о восприятии шрифтов;
- принципы сочетания шрифтов;
- короткие processing principles, извлечённые из skill без раскрытия полной инструкции.

## Базовая структура chunk

```json
{
  "chunk_id": "string",
  "source_type": "spreadsheet | skill_distilled",
  "source_name": "string",
  "source_sheet": "string | null",
  "chunk_type": "font_perception | font_pairing_principle | processing_principle",
  "content": "string",
  "tags": ["string"],
  "public_safe": true
}
```

## Типы chunk

### 1. processing_principle

Короткий принцип обработки запроса, извлечённый из skill без публикации полного skill.

Используется для настройки рамки ответа:
- шрифт как инструмент восприятия;
- сначала задача, потом название шрифта;
- второй шрифт только при наличии роли;
- обязательная проверка кириллицы;
- запрет неподтверждённых платформенных утверждений.

### 2. font_perception

Chunk из листа `Карта шрифтов`.

Описывает тип или параметр шрифта:
- механизм восприятия;
- что считывается;
- примеры Google Fonts;
- риски.

### 3. font_pairing_principle

Chunk из листа `Сочетания шрифтов`.

Описывает принцип шрифтовой пары:
- механизм сочетания;
- рабочая формула;
- хорошие направления;
- риск;
- антипример.

## Как используется в RAG

```text
Запрос пользователя
→ извлечение параметров проекта
→ retrieval query
→ поиск top-k chunks
→ backend собирает prompt из найденных chunks
→ LLM API формирует рекомендацию
```

Модель не обращается к базе сама. Она получает только найденный контекст.
